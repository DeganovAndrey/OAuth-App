### Silent refresh

AccesToken имеет срок жизни, при прохождении определенного времени он протухает. Пользователь не должен видеть как происходит обновление рефреш токена, поэтому он фоново обновляется.

### Флоу Silent refresh

- Запрос уходит с accessToken в заголовке
- сервер возвращает 401 - значит токен протух
- axios interceptor перехватывает 401
- ставится флаг isRefreshing в положение true, и блокаются потворные refresh
- уходит запрос на /api/auth/refresh с refreshToken
- если успех, то новый accessToken сохраняется в памяти, исходный запрос повторяется
- если ошибка, то токены очищаются и редирект на /login
